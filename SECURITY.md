# Security Policy

## Supported Versions

The following versions of OpenClutch are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of OpenClutch seriously. If you believe you've found a security vulnerability, please follow these steps:

### Please DO NOT:
- Open a public GitHub issue
- Post about the vulnerability in public forums, Discord, or social media
- Submit a pull request that fixes the vulnerability (this reveals the issue publicly)

### Please DO:
- Email us directly at **security@openclutch.dev**
- Include a detailed description of the vulnerability
- Provide steps to reproduce the issue
- If possible, suggest a fix or mitigation approach
- Allow us reasonable time to respond and address the issue before any public disclosure

## Response Timeline

When you report a vulnerability, you can expect the following response timeline:

| Timeframe | Action |
|-----------|--------|
| Within 48 hours | Acknowledgment of your report |
| Within 7 days | Initial assessment and classification |
| Within 30 days | Fix developed and tested (or timeline communicated) |
| Upon release | Public disclosure with credit (if desired) |

## Security Measures

OpenClutch implements the following security practices:

- All code changes are reviewed before merging
- Dependencies are regularly audited and updated
- Secrets are never committed to the repository
- Communication between components uses authenticated channels where applicable

## Acknowledgments

We appreciate the security researchers and community members who help keep OpenClutch secure. If you report a valid security issue, we will acknowledge your contribution (with your permission) in our release notes.

## Contact

For security-related questions or concerns, contact: **security@openclutch.dev**
